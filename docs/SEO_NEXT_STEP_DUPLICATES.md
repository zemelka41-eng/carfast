# Следующий SEO-шаг: устранение дублей (Яндекс DUPLICATE)

## Цель

Устранить группы URL, которые Яндекс помечает как **DUPLICATE**, чтобы они не отдавали 200 без необходимости.

## Задачи (отдельная задача)

1. **Собрать список URL-групп**, которые Яндекс помечает как DUPLICATE:
   - в первую очередь: `/product-category/...`, `/бренд/...` (если такие маршруты есть или были);
   - при необходимости — выгрузка из Вебмастера / отчёт `seo_audit_shacman` (секция Duplicate).

2. **Принять решение по каждой группе:**
   - **301** на новые посадочные (каталог/series/category/product);
   - **301** на `/parts/` или другой статический раздел (если уместно);
   - **410 Gone** для «мусорных»/устаревших шаблонов.

3. **Реализовать политику:**
   - в **nginx** (location + return 301/410) — предпочтительно для статических путей;
   - или во **Django** (middleware/view + redirect/HttpResponseGone).

4. **Добавить smoke-проверки:**
   - запрос к старым URL → ожидаемый **301** (Location: новый URL) или **410**;
   - скрипт `scripts/smoke_seo.sh` или отдельный `scripts/check_redirects.sh`.

## Приёмка

- Старые «мусорные» группы URL **не отдают 200** без необходимости (301 или 410).
- Smoke-проверки зелёные.
